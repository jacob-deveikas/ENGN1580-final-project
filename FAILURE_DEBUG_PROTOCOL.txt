FAILURE DEBUG PROTOCOL
======================

If any graded test gives Pe around 0.5 while sync is nonzero, do this first:

1. On BOTH laptops:
python3 bitstream_check.py --n-bits 512 --seed 1580

The bit_hash and first64 must match. If they do not, you are not using the same package/seed.

2. Verify device path with sine:
Receiver:
python3 live_monitor.py --device 1 --ylim 0.2
Transmitter:
python3 interference_generator.py --kind sine --freq 3000 --duration 10 --gain 0.05 --device 1

3. Re-run receiver with --verbose and copy the run folder path.

4. Diagnose the capture:
FSK example:
python3 diagnose_capture.py --wav-in runs/RUN_FOLDER/rx_fsk_capture.wav --mode fsk --rate 500 --n-bits 512 --profiles high

QPSK example:
python3 diagnose_capture.py --wav-in runs/RUN_FOLDER/rx_qpsk_capture.wav --mode qpsk --rate 500 --n-bits 512 --carrier 10400

5. Send these back:
- command lines from both laptops
- bitstream_check output from both laptops
- rx_*_capture.wav
- rx_*_result.json
- tx_*_meta.json from transmitter laptop
- channel_response.json if using measured settings

Do not randomly change gain/profile/carrier until this protocol is done.
